Module Specification Hashes

A Brief on The Concept of Specification Hashes

Some modules in DarkCPU, such as libc or the display drivers, are of some significance to other applications running over the kernel. They need to be recognized and a standard interface needs to be implemented by them. For example, libc must expose a standard interface that can be recognized and used by all applications running overtop of DarkCPU. The LEM graphics display driver needs to implement some routines according to a specific specification that can be recognized and used by all graphics libraries.

DarkCPU's kernel associates all modules with a unique identification code, reffered to as the MD5SpecHash. The MD5SpecHash is defined in the PE Header of an executable image Executable Image Format, If an executable module does not serve other applications or does not need to be recognized by other applications, the MD5SpecHash can be null.

The MD5SpecHash is specific to all implementation of a specific specifications used in DarkCPU. The base specification implemented by that module is defined using the MD5SpecHash. MD5SpecHashes are also used to define extention specifications, that is extentions to a base specification. If a specification is extended, it must be done such that if a user is unaware of this extention, i.e it functions exactly the same as if it had just the base specification implemented.

SpecTable

All MD5SpecHashes should be 128bit MD5 hashes of the following structure to minimize the possibility of a hash collision:

struct SpecTable
{
char sFunction[20];
char sSpecificationName[20];
char sSpecificationPublisher[20];
char bMajorVersion;
char bMinorVersion;
}

SpecTable Description

Name Description
sFunction Describes the function of the module implementing the specification.
sSpecificationName Name of the specification implemented by the module
sSpecificationPublisher Name of the person which published the specification
bMajorVersion Major version of the specification
bMinorVersion Minor version of the specification

Standard Functions

Here is a list of standard SpecTable functions and their descriptions:

Function Description
MediaStorage Standard function name for storage devices.
VFSFSI Standard function name for VFS filesystem interface drivers.
user_mod Standard function name for user libraries modules.
