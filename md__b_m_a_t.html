BitMap Allocation Table File System (BMAT)

VirtualFileSystem

Purpose

The BMAT filesystem serves as a primary read filesystem.

Purpose of Filesystem:

  • Keep the filesystem as simple & fast as possible while allowing for efficient write operations.
  • Maintain a straightforward objective, which is to maintain only a system of files, and to allow the containing storage media to act as the primary boot device.
  • The filesystem is very quickly indexable, with the idea of system memory & slow read times in mind.

Due to the purpose:

  • All data (including file data sections) is aligned in words.
  • File properties remain as primitive as possible.

Directory Flats:

A directory flat is a entire sector consisting of a list of directory entries which describe the filesystem. At the end of every directory flat is a special reserved directory entry referred to as a jump, which links directory flats together.

Directory Entry Flags:

Directory Entry Flag Value Descrition
JUMP 1 Symbolizes a jump type directory entry
TERMINATING 2 Symbolizes the end of the list of file descriptors
DIRECTORY 8 The directory entry describes a firectory

Definitions:

Names Value Descrition
RSFS_ENTRYTYPE_FILE 1 Value of the wType field in the File/Directory entry when the entry describes a file.
RSFS_ENTRYTYPE_DIRECTORY 2 Value of the wType field in the File/Directory entry when the entry describes a directory.
RSFS_HEADER_SIZE 2 The size, in words, of the superblock header.
RSFS_NAME_SIZE 5 The size, in words, of the name field of a File entry.
RSFS_FILEENTRY_SIZE 8 The total size of a file entry, in words.
RSFS_DIRECTORYENTRY_SIZE 8 The total size of a directory entry, in words.

Super-Block

Offset (WORDS) Name Description
0 wSpecVersion Version number of the RSFS spec implementation

File Entry

Offset (WORDS) Name Description
0 wType Entry Type equal to RSFS_FILENTRY_TYPE_FILE
1 fileName Name of the file, including the directory (eg folder/subfolder/myfile.txt) In multi-byte ascii format. Null terminated. 5 words in size
6 wSize The size of the file, in bytes
7 wDataOffset offset from the ramdisk origin where the data for this file resides, in words

Directory Entry

Offset (WORDS) Name Description
0 wType Entry Type equal to RSFS_FILENTRY_TYPE_DIRECTORY
1 folderName Name of the file, including the directory (eg folder/subfolder/myfile.txt) In multi-byte ascii format. Null terminated. 5 words in size
6 wNumEntries Number of entries in the directory.
7 wReserved Padding for fast indexing
