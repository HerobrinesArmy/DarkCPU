Detailed Description

This module is responsible for hadling a large portion of the multithreading in the operating system.

This subsystem performs threading operations such as creating or destorying threads, halting threads, changing a thread's context and swapping threads on interrupt triggers.

void threading_endCycle ( void  )

Ends the current thread's cycle, and passes execution on to the next thread in the chain.

Note
Non-standard calling convention, all registers & stack data are preserved.

Ends the threads current cycle and passes execution on to the next chain. Internally, this just contains the _threading_endCycle() macro followed by a return. If possible, the _threading_endCycle() macro should be used instead to reduce overhead. If the threading system is locked when this function is called, it is automatically freed.

void threading_endOfThread ( void  )

Routine a thread returns to once it returns from its entrypoint. This routine destroys the calling thread.

Note
Uses naked calling convention. nakedCall

Once a thread has finished performing its task, and returns from its path of execution, it will return to this function. Here the thread will be destroyed.

Warning
This routine counts on the fact that the returning thread has restored the stackpointer value to its initial value (the value it was when at when the thread was created.) If the stack has been corrupted, this routine cannot be used.

void threading_interrupt ( void  )

Invoked by the ClockDriver on equal intervals. When invoked, this routine is tasked with the procedure of determining if control should be passed to the next thread, and passing control if it should.

Note
Uses naked-isr calling convention. naked-isr
Warning
This routine must not alter any registers it does not have the means to restore, with the exception of the 'A' register. It must also re-enable interrupts and disable the interrupt queue before passing control over to the next thread.
void threading_lock ( void  )

Locks the threading system. Refer to LockingTheThreadManager.

Note
Uses cdecl calling convention. cdecl

Locks the threading manager such that the calling thread is the only thread executed on the system until the threading system is freed. Regier to LockingTheThreadManager

Returns threading_removeThread ( pThreadContext  )

Removes a given thread from the chain and destroys it.

Note
Uses cdecl calling convention. cdecl

Removes a specified thread from the linked list of threads, and deallocates its ThreadContext structure. If this thread is the active thread, the thread is removed and control is passed on to the next thread in the list.

Parameters
pThreadContextA pointer to the threads ThreadContext structure, otherwise known as the thread's ID.
Returns
Returns 0 on error, and anything else otherwise.
