Standard Driver Interfaces
Note
This document needs improvement.

Purpose

The document provides an overview of driver interfaces implemented by the DarkCPU team, or other well accepted interfaces that are widely-used about the DarkCPU Development Community.

Storage Devices

DarkDev Std Storage Interface (DDSSI)

This is the standard storage device interface. If possible, the device should also implement the DarkDev Std Cachable Storage Extention Interface (DDSCS) as well.

struct mediaStorageDriver
{
sFunction = "mdastg" ;Media storage
sSpecificationName "ddssi0" ;std storage
sSpecificationPublisher = "drkdev" ;
wSpecVersionMajor = 1
wSpecVersionMinor = 0
}

Extentions

DarkDev Std Cachable Storage Extention Interface (DDSCS)

This is the standard extention interface for cachable storage devices. Implementing this will allow more optimal use of your storage device

struct mediaStorageDriver
{
sSpecificationName "ddscs0" ;std Cache
sSpecificationPublisher = "drkdev" ;
wSpecVersionMajor = 1
wSpecVersionMinor = 0
}

Filesystem Interface

DarkDev Std Filesystem Interface (DDSFSI)

struct fsInterfaceDriver
{
sFunction = "vfsfsi" ;fsi = file system interface, vfs = virtual file system
sSpecificationName "ddsfsi" ;fsi = file system interface, vfs = virtual file system
sSpecificationPublisher = "drkdev" ;Dark dev team
wSpecVersionMajor = 1 ;1.0
wSpecVersionMinor = 0
}
